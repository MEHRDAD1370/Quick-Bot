    <article class="markdown-body entry-content" itemprop="text"><h2><a id="user-content-telegram-messenger-cli-" class="anchor" href="#telegram-messenger-cli-" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Telegram messenger CLI <a href="https://travis-ci.org/vysheng/tg"><img src="https://camo.githubusercontent.com/a02f00847e5846240bddb57b4bee1ac46e37a3b7/68747470733a2f2f7472617669732d63692e6f72672f76797368656e672f74672e706e67" alt="Build Status" data-canonical-src="https://travis-ci.org/vysheng/tg.png" style="max-width:100%;"></a></h2>
<p>Command-line interface for <a href="http://telegram.org">Telegram</a>. Uses readline interface.</p>
<h3><a id="user-content-api-protocol-documentation" class="anchor" href="#api-protocol-documentation" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>API, Protocol documentation</h3>
<p>Documentation for Telegram API is available here: <a href="http://core.telegram.org/api">http://core.telegram.org/api</a></p>
<p>Documentation for MTproto protocol is available here: <a href="http://core.telegram.org/mtproto">http://core.telegram.org/mtproto</a></p>
<h3><a id="user-content-upgrading-to-version-10" class="anchor" href="#upgrading-to-version-10" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Upgrading to version 1.0</h3>
<p>First of all, the binary is now in ./bin folder and is named telegram-cli. So be careful, not to use old binary.</p>
<p>Second, config folder is now ${HOME}/.telegram-cli</p>
<p>Third, database is not compatible with older versions, so you'll have to login again.</p>
<p>Fourth, in peer_name '#' are substitued to '@'. (Not applied to appending of '#%d' in case of two peers having same name).</p>
<h3><a id="user-content-installation" class="anchor" href="#installation" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Installation</h3>
<p>Clone GitHub Repository</p>
<pre><code> git clone --recursive https://github.com/vysheng/tg.git &amp;&amp; cd tg
</code></pre>
<h3><a id="user-content-python-support" class="anchor" href="#python-support" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Python Support</h3>
<p>Python support is currently limited to Python 2.7 or Python 3.1+. Other versions may work but are not tested.</p>
<h4><a id="user-content-linux-and-bsds" class="anchor" href="#linux-and-bsds" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Linux and BSDs</h4>
<p>Install libs: readline, openssl and (if you want to use config) libconfig, liblua, python and libjansson.
If you do not want to use them pass options --disable-libconfig, --disable-liblua, --disable-python and --disable-json respectively.</p>
<p>On Ubuntu/Debian use:</p>
<pre><code> sudo apt-get install libreadline-dev libconfig-dev libssl-dev lua5.2 liblua5.2-dev libevent-dev libjansson-dev libpython-dev make 
</code></pre>
<p>On gentoo:</p>
<pre><code> sudo emerge -av sys-libs/readline dev-libs/libconfig dev-libs/openssl dev-lang/lua dev-libs/libevent dev-libs/jansson dev-lang/python
</code></pre>
<p>On Fedora:</p>
<pre><code> sudo dnf install lua-devel openssl-devel libconfig-devel readline-devel libevent-devel libjansson-devel python-devel
</code></pre>
<p>On Archlinux:</p>
<pre><code> yaourt -S telegram-cli-git
</code></pre>
<p>On FreeBSD:</p>
<pre><code> pkg install libconfig libexecinfo lua52 python
</code></pre>
<p>On OpenBSD:</p>
<pre><code> pkg_add libconfig libexecinfo lua python
</code></pre>
<p>On openSUSE:</p>
<pre><code> sudo zypper in lua-devel libconfig-devel readline-devel libevent-devel libjansson-devel python-devel libopenssl-devel
</code></pre>
<p>Then,</p>
<pre><code> ./configure
 make
</code></pre>
<h4><a id="user-content-other-methods-to-install-on-linux" class="anchor" href="#other-methods-to-install-on-linux" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Other methods to install on linux</h4>
<p>On Gentoo: use ebuild provided.</p>
<p>On Arch: <a href="https://aur.archlinux.org/packages/telegram-cli-git">https://aur.archlinux.org/packages/telegram-cli-git</a></p>
<h4><a id="user-content-mac-os-x" class="anchor" href="#mac-os-x" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Mac OS X</h4>
<p>The client depends on <a href="http://cnswww.cns.cwru.edu/php/chet/readline/rltop.html">readline library</a> and <a href="http://www.hyperrealm.com/libconfig/">libconfig</a>, which are not included in OS X by default. You have to install these libraries manually.</p>
<p>If using <a href="http://brew.sh/">Homebrew</a>:</p>
<pre><code> brew install libconfig readline lua python libevent jansson
 export CFLAGS="-I/usr/local/include -I/usr/local/Cellar/readline/6.3.8/include"
 export LDFLAGS="-L/usr/local/lib -L/usr/local/Cellar/readline/6.3.8/lib"
 ./configure &amp;&amp; make
</code></pre>
<p>Thanks to <a href="https://github.com/vysheng/tg/issues/3#issuecomment-28293731">@jfontan</a> for this solution.</p>
<p>If using <a href="https://www.macports.org">MacPorts</a>:</p>
<pre><code> sudo port install libconfig-hr
 sudo port install readline
 sudo port install lua51
 sudo port install python34
 sudo port install libevent
 export CFLAGS="-I/usr/local/include -I/opt/local/include -I/opt/local/include/lua-5.1"
 export LDFLAGS="-L/usr/local/lib -L/opt/local/lib -L/opt/local/lib/lua-5.1"
 ./configure &amp;&amp; make
</code></pre>
<p>Install these ports:</p>
<ul>
<li>devel/libconfig</li>
<li>devel/libexecinfo</li>

<pre><code>bin/telegram-cli -k &lt;public-server-key&gt;
</code></pre>
<p>By default, the public key is stored in tg-server.pub in the same folder or in /etc/telegram-cli/server.pub. If not, specify where to find it:</p>
<pre><code>bin/telegram-cli -k tg-server.pub
</code></pre>
<p>Client support TAB completion and command history.</p>
<p>Peer refers to the name of the contact or dialog and can be accessed by TAB completion.
For user contacts peer name is Name  Lastname with all spaces changed to underscores.
For chats it is it's title with all spaces changed to underscores
For encrypted chats it is &lt;Exсlamation mark&gt;  Name  Lastname with all spaces changed to underscores.</p>
<p>If two or more peers have same name, number is appended to the name. (for example A_B, A_B#1, A_B#2 and so on)</p>
